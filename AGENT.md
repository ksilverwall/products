# AGENT MEMO

ユーザーからの指示のうち、HTMLを読むだけでは汲み取れない運用ルールを記録する。

## 変更ルール
- 連絡先（メール / フォーム / 電話方針）を更新するときは、トップページと X Auto Follow 配下の全ページを一斉に差し替える。ページによって表記揺れを残さないこと。
- 電話番号は原則非公開。必要時にメール／フォーム経由で請求されたユーザーへ個別通知する運用を継続する。
- 特商法や各ポリシーの最終更新日は同じ日付で揃え、情報を更新したタイミングで必ず修正する。
- データ削除の連絡窓口は「Googleフォームが原則／メールは代替手段」という方針を維持する。フォーム側にも削除依頼メニューを用意して運用する。
- Stripe 審査を常に想定し、法的表示・支払い条件・返金ポリシーなどは実態と差異が出たら即時反映する。

## 情報設計
- リポジトリ直下の `index.html` はブランド全体のランディングページ。新しいプロダクトを追加する場合はカードを増やし、対応するポリシーリンクを追記する。
- 各ページにトップへ戻る導線（`../index.html` など）とパンくずを設置しておく。新しいページを作る際も同じ導線を忘れずに入れる。
- HTML はスタンドアロンで運用しているため、ビルドレス前提（追加ライブラリなし）。レイアウト調整はインライン CSS で完結させる。
